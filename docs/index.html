REST APIs from route definitions

Create application APIs with `the-api`

`the-api` is a module for building Hono-based REST APIs backed by PostgreSQL. Define tables, joins, validation, permissions, files, and relations once, then get consistent CRUD routes and response envelopes across applications.

Before you start

Prerequisites

Node.js or Bun

Use Node.js 18+ or Bun 1+.

PostgreSQL

The database must be running and reachable from the API.

`the-api` installed

Install it in each application that uses it as a module.

Tables and migrations

Create tables before startup, usually through migrations.

Install

Package and environment

Install the module

bun add the-api

npm install the-api

Example data model

Tables used in the examples

These examples use `ships`, `categories`, `users`, `images`, `regions`, `lang`, and `ship_owners`. Quoted identifiers preserve camelCase fields in PostgreSQL. 

CREATE TABLE categories (
  id SERIAL PRIMARY KEY,
  name TEXT NOT NULL
);

CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  email TEXT NOT NULL,
  "companyName" TEXT,
  roles TEXT[] DEFAULT ARRAY[]::TEXT[]
);

CREATE TABLE ships (
  id SERIAL PRIMARY KEY,
  name TEXT NOT NULL,
  "categoryId" INTEGER REFERENCES categories(id),
  "userId" INTEGER REFERENCES users(id),
  price NUMERIC,
  status TEXT DEFAULT 'draft',
  sold BOOLEAN DEFAULT false,
  deleted BOOLEAN DEFAULT false,
  "externalId" TEXT,
  "externalUrl" TEXT,
  "timeCreated" TIMESTAMP DEFAULT now(),
  "timeUpdated" TIMESTAMP,
  "timeDeleted" TIMESTAMP
);

CREATE TABLE images (
  id SERIAL PRIMARY KEY,
  "shipId" INTEGER REFERENCES ships(id),
  name TEXT NOT NULL,
  "originalName" TEXT,
  path TEXT NOT NULL,
  "isMain" BOOLEAN DEFAULT false
);

Quickstart

Build the smallest useful API

import { Routings, TheAPI, middlewares } from 'the-api';

const router = new Routings({ migrationDirs: ['./migrations'] });

router.crud({ table: 'ships' });

const theAPI = new TheAPI({
  routings: [middlewares.common, router],
});

await theAPI.up();

Query Meaning Example
_fieldsSelect response fields._fields=id,name,price
_sortSort ascending or descending._sort=-price,-id
_limit, _page, _skipOffset pagination._limit=20&_page=2
_afterCursor pagination._after=...
_searchFuzzy search by `searchFields`._search=Auro
field=valueEquality filter.status=published
field!=valueNot equal.status!=archived

Joins

Embed related data

router.crud({
  table: 'ships',
  join: [
    {
      table: 'categories',
      where: '"categories"."id" = "ships"."categoryId"',
    },
  ],
});
router.crud({
  table: 'ships',
  join: [
    {
      table: 'images',
      where: '"ships"."id" = "images"."shipId"',
      fields: ['id', 'name', 'originalName', 'path'],
      orderBy: '"isMain" DESC, "id" ASC',
      limit: 1,
    },
  ],
});
router.crud({
  table: 'ships',
  joinOnDemand: [
    {
      table: 'images',
      alias: 'images',
      where: '"ships"."id" = "images"."shipId"',
      fields: ['id', 'path'],
    },
  ],
});
curl 'http://localhost:7788/ships/1?_join=images&_fields=id,name,images'

Fields

Hide, lock, expose, and edit fields by permission

router.crud({
  table: 'ships',
  fieldRules: {
    hidden: ['externalId', 'externalUrl'],
    readOnly: ['id', 'userId', 'timeCreated', 'timeUpdated', 'deleted', 'status'],
    visibleFor: {
      'ships.getInternalInfo': ['externalId', 'externalUrl'],
    },
    editableFor: {
      'ships.editStatus': ['status'],
    },
  },
});
Strict body whitelists need custom validation. Automatic validation checks known fields, but it does not reject every unknown body key by itself.

Roles

Protect CRUD methods

import Roles from 'the-api-roles';

const roles = new Roles({
  root: ['*'],
  admin: ['ships.*'],
  registered: ['ships.get'],
  guest: ['ships.get'],
  owner: ['ships.patch', 'ships.delete', 'ships.getInternalInfo'],
});

router.crud({
  table: 'ships',
  permissions: {
    methods: ['GET', 'POST', 'PATCH', 'DELETE'],
    owner: ['ships.getInternalInfo'],
  },
});

Permission names use <prefix-or-table>.<methodLowerCase>, for example ships.get, ships.post, ships.patch, and ships.delete. Requests without a token are treated as guest; tokens without roles are not.

Validation

Automatic and custom validation

CRUD validation is generated from PostgreSQL introspection by default. Override or extend it with `validation`. 

router.crud({
  table: 'ships',
  validation: {
    body: {
      post: {
        name: { type: 'string', required: true },
        categoryId: { type: 'number', required: true },
        price: { type: 'number', min: 0 },
      },
      patch: {
        price: { type: 'number', min: 0 },
      },
    },
  },
});
RuleMeaning
type`string`, `number`, `boolean`, `date`, `enum`, `array`, `object`, or an array of types.
requiredField must be present and not empty.
enumAllowed values.
min, maxNumeric bounds.
preprocessTransform value before validation.
items, propertiesNested array/object schemas.

Read and define errors

Register

    code: 1001,
    status: 409,
    description: 'Ship is already sold',

throw new Error('SHIP_ALREADY_SOLD');

    "status": 409,
    "description": "Ship is already sold",
    "name": "SHIP_ALREADY_SOLD",
    "additional": [],
    "error": true

}

NOT_FOUND404No route or result.
INVALID_TOKEN401JWT verification failed.

Files

Local and object-storage uploads

Local

FILES_FOLDER=public/files
IMAGE_SIZES=thumb:200x200,large:1200x800

MinIO or S3-compatible

MINIO_ENDPOINT=minio.example.com
MINIO_PORT=9000
MINIO_USE_SSL=true
MINIO_ACCESS_KEY=access-key
MINIO_SECRET_KEY=secret-key
MINIO_BUCKET_NAME=prod
router.post('/ships/:id/images', async (c) => {
  const files = await c.var.files.uploadBody(
    c.var.body as Record<string, unknown>,
    `ships/${c.req.param('id')}`,
    { fields: ['file', 'file[]'], imagesOnly: true },
  );

  c.set('result', files);
});

const theAPI = new TheAPI({
  routings: [middlewares.common, middlewares.files, router],
});

Relations

Return normalized related data

router.crud({
  table: 'ships',
  relations: {
    categoryId: {
      table: 'categories',
      relationIdName: 'id',
    },
    userId: {
      table: 'users',
      relationIdName: 'id',
      fieldRules: {
        hidden: ['email'],
      },
    },
  },
});
{
  "result": [
    {
      "id": 1,
      "name": "Aurora",
      "categoryId": 1,
      "userId": 1
    }
  ],
  "relations": {
    "categoryId": {
      "1": { "id": 1, "name": "Cargo" }
    },
    "userId": {
      "1": { "id": 1, "companyName": "Blue Ocean Ltd" }
    }
  },
  "error": false
}

Use `join` when related data should be embedded in each result. Use `relations` when clients benefit from normalized, deduplicated data.

db, dbWriteKnex read/write connections.

router.get('/my-ships', async (c, next) => {
  c.var.appendQueryParams({
    userId: c.var.user.userId as number,
    deleted: false,
  });
  await next();
});

Middleware

Built-in and Hono middleware

import {
  Routings,
  TheAPI,
  cors,
  csrf,
  compress,
  etag,
  bodyLimit,
  middlewares,
} from 'the-api';

const theAPI = new TheAPI({
  routings: [
    cors(),
    csrf(),
    compress(),
    etag(),
    bodyLimit({ maxSize: 5 * 1024 * 1024 }),
    middlewares.common,
    middlewares.info,
    middlewares.files,
    middlewares.email,
    router,
  ],
});

`middlewares.common` is `[logs, errors, status]` and is usually placed before application routes.

Project work

Testing and building `the-api`

docker compose -f tests/docker-compose.yml up -d
bun --env-file=.env.test test
bun test tests/crud/simple.spec.ts
bun run build

Build output goes to `dist/`. Change `src/`, then run the focused tests and `bun run build`; do not edit `dist/` by hand.
